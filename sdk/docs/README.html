INSTALLATION:

NOTE: With new installations, you might be asked to activate Windows and download and execute Windows Installer 3.0 etc.

01: Execute dotnetfx3setup.exe and accept all defaults.

02: Execute Microsoft WSE Runtime and accept all defaults.

03: Execute apache1.1.37-win32-x86-nosrc.msi and accept all defaults.

04: Execute JDK-150_11-windows-i586-p.exe and accept all defaults.

05: Execute mysql-connector-odbc-3.5.15-win32.msi and accept all defaults.

06: Execute mysql-5.0.41-win32 and accept all defaults for the installation part. At the end of installation part, it will start MySQL Server Configuration process. Select defaults except for following options (select the following choices when presented):

a) Detailed Configuration
b) Developer Machine (Recommended)
c) Transactional Database Only
d) Best Support For Multilingualism
e) Install As Windows Service (automatic launch)
f) Enter root password and WRITE IT DOWN as you will need it later.

07: Extract and copy all time zone tables in Program Files\mysql\mysql server\data\mysql.

08: Execute php-5.2.2-win32-installer.msi. Accept defaults except following options:

NOTE: If any step is missed or incorrect, use this installer to uninstall and install correctly.

a) Select Apache 1.3.x Module

b) Specify Apache Configuration Directory which is typically: c:\Program Files\Apache Group\Apache\conf

c) Under the "Choose Items to Install"

i) Under Program menu, select CLI Executable and Script Executable
ii) Under Extensions menu, select:
    - bzip2
    - LDAP
    - MySQL
    - MySQLi
    - Sockets
    - XML- RPC (If available)
iii) Under Extras menu, select PEAR

09: Execute C:\Program Files\PHP\go-pear.bat and select following options:

- system Wide or local: system
- all or Enter to continue: Hit Enter
- alter php.ini: Y
- accept others as default or hit enter to continue
- if this process suggests, run the registry file it creates.

10: Run the command "pear install --alldeps " till all the following packages are installed. Please note --aldeps option will install one or more of the following packages at one go (for example, it will auto download and install NetURL and NetSocket as these are required for the HTTP_Request). Sometime it will specify a specific package name - simply reissue the "pear install" command with specified packege name:

- HTTP_Request
- Mail
- Mail_Mime
- Net_DIME
- Net_Socket
- Net_URL
- PEAR
- SOAP(-beta)
- XML_Parser
- XML_RPC
- Net_SMTP

Issuing the command "pear list" should show the following:

- Archive_Tar
- Console_Getopt
- HTTP_Request
- Mail
- Mail_Mime
- Net_DIME
- Net_Socket
- Net_URL
- PEAR
- SOAP
- XML_Parser
- XML_RPC
- Net_SMTP

11: HMP: Execute setup.exe from HMP directory. Accept defaults and select top 4 installation checkboxes.

12: NEOSPEECH:

Extract the MASTER and TTS_SERVER zips downloaded earlier and run two Setup.exe files and accept all defaults. After the installation edit c:\ttssrv.ini file to change NfsDir path to match CUME path - C:/Program Files/Cisco Systems/Unified Application Environment/MediaServer. Also, verify following two lines are present:

KATE SELECT 1

YUMI SELECT 0

Restart your machine.

(A CUAE application might fail to play TTS first time, but plays on second time onwards)

13: [VERIFY] Comment out the line starting with "PHPIniDir" in C:\Program Files\Apache Group\Apache\conf\httpd.conf file (Or use your apache installation specific path). This step applies to Apache 1.3.X versions.

14: [VERIFY] Uncomment "AddModule moduniqueid.c" and "LoadModule uniqueidmodule modules/moduniqueid.so" in C:\Program Files\Apache Group\Apache\conf\httpd.conf file (Or use your apache installation specific path).

15: [VERIFY] In C:\Program Files\PHP\php.ini file uncomment "errorreporting = EALL & ~ENOTICE" line and comment out "errorreporting = E_ALL" line.

16: At the bottom of C:\Program Files\Apache Group\Apache\conf\httpd.conf file add the following line:

Include "c:\Program Files\Cisco Sysems\Unified Application Environment\System\Apache\conf"

17: Restart apache (net stop apache && net start apache).

NOTE: FOLLOWING 3 STEPS WOULD NEED ROOT/PASSWORD FROM STEP 6f above.

18: Install CUAE Platform Services. Accept defaults.

19: Install CUAE Application Server. Accept defaults.

20: Install CUAE Media Engine. Accept defaults.

21: RESTART MACHINE.

NOTE: FOLLOWING STEPS ARE SPECIFIC TO APPSUITEADMIN EXAMPLES INSTALLATION

22: Install CUAE SDK installer and accept defaults.

23: Edit initialize_db.bat from C:\Program Files\Cisco Unified Application Environment SDK\sdk\examples\appsuite\src\web\appsuiteadmin\install directory to indicate correct MySQL installation path info and user/password info. It uses root\metreos as default user/pass, but it accepts these as arguments if there is a need to override. After modification, save and run it.

24: Copy C:\Program Files\Cisco Unified Application Environment SDK\sdk\examples\appsuite\src\web\appsuiteadmin\install\apache\conf\appsuite.conf to c:\Program Files\Cisco Sysems\Unified Application Environment\System\Apache\conf. Edit this appsuite.conf file to indicate correct appsuiteadmin direcotry locations in multiple places in this file, which by default is C:\Program Files\Cisco Unified Application Environment SDK\sdk\examples\appsuite\src\web\appsuiteadmin. Also, edit to indicate correct PEAR location which is C:\Program Files\PHP\PEAR. NOTE: USE FORWARD SLASHES.

25: Edit C:\Program Files\Cisco Unified Application Environment SDK\sdk\examples\appsuite\src\web\appsuiteadmin\includes\class.ReplicationSettingsManagement.php file and change the first

26: Make sure C:\Program Files\PHP\php.ini file's session.save_path variable is pointing to a valid path and uncommented.

27: RESTART MACHINE.
