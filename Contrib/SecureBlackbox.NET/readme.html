Installation

Welcome to SecureBlackbox. You have just installed .NET component library.
Usually the files are integrated into Visual Studio, Borland C#Builder and Borland Delphi for .NET automatically. If you need to add the components to Toolbox manually, do the following:

In Visual Studio 2005

  1. Use Main Menu -> Tools -> Choose Toolbox items to open Toolbox Customization dialog
  2. In the dialog that appears activate .NET Framework Components tab
  3. Use a filter to show only those components, which contain "SecureBlackbox" in the assembly name. To do this put "SecureBlackbox" to the filter box.
  4. Check all components that appear.
  5. If you don't find some component in the list, use Browse button to find and add the assembly, which contains the component, to the list

In Visual Studio.NET 2003

  1. Use Main Menu -> Tools -> Add / Remove Toolbox Items to open Toolbox Customization dialog
  2. In the dialog that appears activate .NET Framework Components tab
  3. Sort the list by name.
  4. Search the list for all components, which start with "TEl" and are located in SecureBlackbox assemblies.
  5. Check all components that you find.
  6. If you don't find some component in the list, use Browse button to find and add the assembly, which contains the component, to the list

In Borland Developer Studio, Delphi for .NET and Borland C#Builder

  1. Use Main Menu -> Component -> Intalled .NET Components to open .NET Components dialog
  2. In the dialog that appears activate .NET Components tab
  3. Sort the list by name.
  4. Search the list for all components, which start with "TEl" and are located in SecureBlackbox assemblies.
  5. Check all components that you find.
  6. If you don't find some component in the list, use Browse button to find and add the assembly, which contains the component, to the list

Activation of evaluation version

SecureBlackbox components require a key, without which they can't be used. For evaluation purposes you will find a trial key in the folder where you installed SecureBlackbox. The key is named "LicenseKey.der". To pass the license key to components, you need to call SetLicenseKey function which is located in SecureBlackbox assembly. All sample projects include this call.

You will find demo projects for C# and VB.NET in <SecureBlackbox>\Samples subfolder.

Additional packages and sample projects

SSLBlackbox includes not only SSL classes themselves, but also components that can be used with third-party libraries. Currently RemObjects SDK for .NET and RemObjects InternetPack are supported. These components are not installed by default. You will need to build and install the packages as described below.

  • RemObjects products:
    You need to install the SSLConnection project located in <SecureBlackbox>\Classes\RemObjects.InternetPack folder.
    Note that you need to have RemObjects.InternetPack 1.0.1.49 and later to use SecureBlackbox.NET.

Deployment

You need to deploy the assemblies that your project refers to.
SecureBlackbox.dll is always used (even when it is not explicitly included into References list).
Note, that some assemblies depend on other assemblies, even if the latter are not explicitly included into the project.

Dependencies are:

  • SecureBlackbox.PKI.dll requires SecureBlackbox.dll.
  • SecureBlackbox.SSLClient.dll requires SecureBlackbox.SSLCommon.dll, SecureBlackbox.PKI.dll and SecureBlackbox.dll.
  • SecureBlackbox.SSLServer.dll requires SecureBlackbox.SSLCommon.dll, SecureBlackbox.PKI.dll and SecureBlackbox.dll.
  • SecureBlackbox.SSLSocket.dll requires SecureBlackbox.SSLCommon.dll, SecureBlackbox.PKI.dll and SecureBlackbox.dll. It also requires SecureBlackbox.SSLClient.dll if client sockets are used and SecureBlackbox.SSLServer.dll if server sockets are used.
  • SecureBlackbox.FTPS.dll requires SecureBlackbox.SSLClient.dll, SecureBlackbox.SSLCommon.dll, SecureBlackbox.PKI.dll and SecureBlackbox.dll.
  • SecureBlackbox.HTTPS.dll requires SecureBlackbox.SSLClient.dll, SecureBlackbox.SSLCommon.dll, SecureBlackbox.PKI.dll and SecureBlackbox.dll.
  • SecureBlackbox.SSHClient.dll requires SecureBlackbox.SSHCommon.dll and SecureBlackbox.dll. It can also require SecureBlackbox.PKI.dll if X.509 Certificates are used.
  • SecureBlackbox.SSHServer.dll requires SecureBlackbox.SSHCommon.dll and SecureBlackbox.dll. It can also require SecureBlackbox.PKI.dll if X.509 Certificates are used.
  • SecureBlackbox.SFTP.dll requires SecureBlackbox.SFTPCommon.dll, SecureBlackbox.SSHCommon.dll, SecureBlackbox.SSHClient.dll and SecureBlackbox.dll. It can also require SecureBlackbox.PKI.dll if X.509 Certificates are used.
  • SecureBlackbox.SFTPServer.dll requires SecureBlackbox.SFTPCommon.dll, SecureBlackbox.SSHCommon.dll, SecureBlackbox.SSHServer.dll and SecureBlackbox.dll. It can also require SecureBlackbox.PKI.dll if X.509 Certificates are used.
  • SecureBlackbox.MIME.dll requires SecureBlackbox.Charsets.dll and SecureBlackbox.dll.
  • SecureBlackbox.SMIME.dll requires SecureBlackbox.PKI.dll, SecureBlackbox.MIME.dll, SecureBlackbox.Charsets.dll and SecureBlackbox.dll.
  • SecureBlackbox.PGP.dll requires SecureBlackbox.PKI.dll and SecureBlackbox.dll.
  • SecureBlackbox.PGPMIME.dll requires SecureBlackbox.PGP.dll, SecureBlackbox.MIME.dll, SecureBlackbox.Charsets.dll and SecureBlackbox.dll. It can also require SecureBlackbox.PKI.dll if X.509 Certificates are used.
  • SecureBlackbox.PDF.dll requires SecureBlackbox.PKI.dll and SecureBlackbox.dll.
  • SecureBlackbox.XML.dll requires SecureBlackbox.PGP.dll, SecureBlackbox.Charsets.dll, SecureBlackbox.PKI.dll and SecureBlackbox.dll.

To deploy SecureBlackbox with your project, use your regular installer tool. DO NOT USE SecureBlackbox installation package (which you downloaded and installed for development) for deployment purposes.
When deploying the project, copy the above described assemblies to your application folder. Questions about when and how to install the assemblies to Global Assembly Cache are discussed in Working with Assemblies and the Global Assembly Cache and How to: Install an Assembly into the Global Assembly Cache articles.

Patents

SecureBlackbox includes support for IDEA (International Data Encryption Algorithm), which is protected under a patent registered in the USA, Europe and Japan. MediaCrypt AG holds all the relevant rights related to the distribution of the IDEA algorithm. Ascom, Switzerland holds the patents.

You choose whether to use IDEA or not. SecureBlackbox provides a way to turn off IDEA support completely. Alternatively, you can aquire licenses from MediaCrypt AG.
IDEA is used in SSL / TLS, SSH and PGP algorithms and protocols. While in SSL / TLS and SSH protocols IDEA support can be turned off with no effect, with PGP the situation is more complicated. IDEA was used as a main encryption algorithm for storing PGP keys in PGP 2.x.. So if you don't use PGP, or you don't need to support PGP 2.x keys, you can safely disable IDEA in SecureBlackbox.

To disable IDEA support in .NET edition, set global variable SBIDEA.Unit.IdeaSupported to false.
To disable IDEA support in VCL edition, set global variable SBIDEA.IdeaSupported to false.
